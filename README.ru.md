# TestBlox

**Автоматизация тестирования без кода с ИИ и Git.** Создавайте, генерируйте и запускайте UI- и API-тесты без программирования. Всё хранится в репозитории, удобно работать в команде.

[![Windows](https://img.shields.io/badge/Windows-x64-blue?logo=windows)](https://github.com/arslan-ahmetjanov/testblox/releases)
[![License](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

## Зачем TestBlox?

Ручная разработка тестов и настройка автоматизации отнимают время. TestBlox объединяет:

- **Без кода** — создание тестов в визуальном редакторе: шаги, элементы и проверки без программирования.
- **Генерация ИИ** — опишите страницу или вставьте URL Swagger/OpenAPI; получите готовые сценарии в один клик.
- **Git из коробки** — тесты живут в репозитории (`.testblox/`): версионность, код-ревью, артефакты для CI.
- **UI и API в одном месте** — один рабочий стол для веб-сценариев и API-проверок с общими переменными и структурой.

Бесплатного десктопного no-code инструмента с **генерацией тестов на ИИ и интеграцией с Git** в одном приложении сегодня нет. TestBlox создан для тестировщиков, которые хотят работать быстро, не выходя из привычного процесса.

## Возможности

| Возможность | Описание |
|-------------|----------|
| **Визуальный конструктор тестов** | Добавляйте шаги, выбирайте элементы с распарсенных страниц, используйте общие шаги и переменные. |
| **Генерация тестов ИИ** | Подключите OpenRouter (или совместимый API); генерируйте UI- и API-сценарии по страницам и Swagger. |
| **Парсер страниц** | Укажите URL; приложение извлечёт кликабельные элементы и построит стабильные селекторы (например, `data-testid`, `data-qa`, `id`). |
| **Swagger/OpenAPI** | Импорт спецификации по URL; получение эндпоинтов и генерация API-сценариев. |
| **Запуск на Playwright** | Запуск UI-тестов в Chromium (в комплекте). Быстро, надёжно, без отдельной установки. |
| **API test runner** | Выполнение HTTP-запросов, проверка статуса и тела (например, JSONPath). |
| **Интеграция с Git** | Откройте папку с Git-репозиторием; коммитьте и пушите тесты вместе с командой. |
| **Отчёты** | Просмотр результатов прогонов и истории в приложении. |

## Требования

- **ОС:** Windows 10/11 (x64)
- **ОЗУ:** минимум 4 ГБ, рекомендуется 8 ГБ
- **Диск:** ~500 МБ для приложения и встроенного Chromium

## Установка

### Вариант 1: Скачать релиз (рекомендуется)

1. Перейдите в [Releases](https://github.com/arslan-ahmetjanov/testblox/releases).
2. Скачайте:
   - **TestBlox Setup 1.0.0.exe** — установщик (рекомендуется), или  
   - **TestBlox 1.0.0.exe** — портативная версия, без установки.
3. Запустите файл. Портативную версию можно запускать из любой папки.

### Вариант 2: Сборка из исходников

```bash
git clone https://github.com/arslan-ahmetjanov/testblox.git
cd testblox
npm ci
npm run build
```

Результат: `dist/TestBlox Setup 1.0.0.exe` и `dist/TestBlox 1.0.0.exe` (портативная).

## Быстрый старт

1. **Открыть или создать рабочее пространство**  
   - *Открыть:* выберите папку (например, репозиторий проекта).  
   - *Создать:* создайте папку и откройте её; TestBlox инициализирует структуру `.testblox`.

2. **Добавить страницу (для UI-тестов)**  
   - В приложении добавьте страницу и введите URL.  
   - Нажмите «Parse», чтобы загрузить страницу; появятся элементы и селекторы.

3. **Создать тесты**  
   - Вручную: добавьте тест, шаги, выберите элементы и действия.  
   - С ИИ: Настройки → укажите API-ключ OpenRouter и модель → «Generate with AI» на странице или «Generate from selection» для страниц и Swagger.

4. **Запустить тесты**  
   - Выберите один или несколько тестов и запустите.  
   - Результаты смотрите в отчёте в приложении.

5. **Версионность**  
   - Закоммитьте папку `.testblox/` (и при необходимости `pages/`, `tests/`) в Git, чтобы команда могла использовать и развивать тесты.

## Настройка ИИ (OpenRouter)

1. Получите API-ключ на [OpenRouter](https://openrouter.ai/).
2. В TestBlox: **Настройки** → **LLM**.
3. Укажите:
   - **API URL:** `https://openrouter.ai/api/v1/chat/completions`
   - **API key:** ваш ключ
   - **Model:** например, `openai/gpt-4o-mini` или другая модель.
4. Сохраните. Доступны «Generate with AI» и «Generate from selection».

**Ключ через переменные окружения:** Можно задать `TESTBLOX_LLM_API_KEY` (и при необходимости `TESTBLOX_LLM_MODEL`, `TESTBLOX_LLM_API_BASE_URL`) в окружении или в файле `.env` в корне workspace. Приложение подгружает `.env` перед вызовом LLM, ключ не сохраняется в Настройках и в `.testblox/llm.json`. В CI задайте эти переменные в пайплайне (GitHub Secrets, GitLab CI variables).

## Структура проекта (рабочее пространство)

При открытии папки как рабочего пространства TestBlox использует (и может создать):

```
ваш-проект/
├── .testblox/          # конфиг рабочего пространства, действия, переменные, общие шаги
├── pages/              # распарсенные страницы (URL, элементы, селекторы)
├── tests/              # тест-кейсы (UI и API)
├── endpoints/          # API-эндпоинты (например, из Swagger)
└── reports/            # отчёты прогонов (часто в .gitignore)
```

Храните в Git `.testblox/`, `pages/`, `tests/` и `endpoints/`, если хотите версионировать тесты вместе с проектом.

## Переменные и секреты

Значения переменных (логины, пароли, токены) хранятся в файле **`.env`** в корне workspace. Файл создаётся при инициализации проекта или при сохранении первой переменной и не коммитится (указан в .gitignore).

- **Переменные без pattern** — имя в `.testblox/variables.json`, значение в `.env`. В шагах и полях auth используйте `{{variableName}}`.
- **Переменные с pattern** (regex) — в variables.json только pattern; значение генерируется при каждом запуске, в .env не записывается.
- **В CI** — задайте те же имена переменных через env пайплайна (GitHub Secrets, GitLab CI variables).
- **Прямая ссылка на env** — в любом поле (например, auth token или password) можно использовать `{{env:VAR_NAME}}` для подстановки из `process.env.VAR_NAME` (из `.env` или env CI).

## Стек технологий

- **Десктоп:** Electron  
- **UI:** React, Vite, Material UI
